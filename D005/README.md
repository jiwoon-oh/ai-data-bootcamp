# 모두마켓 웹 트래픽 — 로그 분석 리포트

## 1. 데이터 개요
- 로그 줄 수: 1500줄
- 기간: 2025-03-01 ~ 2025-03-07
- 추출 필드: ip, ts, method, path, status, size

## 2. 파싱 (어떻게 표로 만들었나)
- 사용한 정규식 패턴의 핵심: IP는 `\d+\.\d+...`, 시각은 `[^\]]+` 로 추출
- 날짜 형식: %d/%b/%Y:%H:%M:%S 로 파싱, 실패 0건

## 3. 트래픽 요약
- 총 요청 수: 1500건
- 오류 응답(4xx·5xx) 비율: 23.2%
- 요청 많은 경로 TOP 3: /, /search, /cart

## 4. 시간 패턴 (리샘플링에서 읽은 것)
- 트래픽이 가장 몰린 시간대: 2025-03-07 22시, 21건
- 트래픽이 가장 많았던 날 : 2025-03-06 (Thursday) - 225건
- 트래픽이 가장 적었던 날 : 2025-03-05 (Wednesday) — 198건

## 5. 전체 평가 및 다음 분석 제안
- 평일에는 주로 오후시간대에 트래픽이 많이 발생을 했고 주말에는 오전에 가장 많은 트래픽이 발생하였다. 
- 한가지 특이한 점은 목요일에 가장 많은 트래픽이 발생한것으로 보아 해당 지역은 목요일에 아마도 공휴일이었던 것으로 사려된다.
- 404오류와 500오류가 전체 상태 중 23.2%의 비율로 발생하였다. 에러율이 가장 높은 상위 3개 경로들의 점검이 필요해보인다. 특히 checkout 페이지에서 404에러가 가장 많이 발생하고 있으므로 라우팅 점검이 권장된다. cart 페이지에서 500이 많이 발생하는 것으로 보아 결제 시 버그가 의심되므로 후속히 처리해야한다. 또한 특정 제품 페이지에서 25.1% 비율로 에러가 발생중이니 해당 제품 페이지를 수정하고 리다이렉션을 고려해보아야 한다.
- 특정 IP에서 높은 오류율이 발생하였다. request가 많은 IP 상위 10개 중 오류율이 20%가 넘는 IP들이 많이 존재하니 공격시도가 의심된다.
