# 옷장패션 데이터 정제 보고서

## 1. 데이터 개요
- 행/열: 1,500 × 8 (id포함)
- 주요 컬럼: order_id, customer_age, category, channel, price, quantity, amount, return_amount

## 2. 진단 결과
- 결측: amount 3.40%, price 0.33%
- amount 결측 행의 분포가 app 88%로 과하게 쏠려있음 (MAR 의심)
- price 결측은 0.3%에 불과, MCAR 가능
- 이상치(IQR): customer_age(999·0, 18건), quantity(200, 1건), amount(50,000,000, 145건)

## 3. 처리 결정과 근거
1. amount 결측 (MAR)
채널별 중앙값으로 대체 | 채널별 중앙값과 평균값을 비교해보았을때 평균값으로 대체하면 mean이 거의 그대로 유지되는것을 볼 수 있다 이는 이상치가 있는 평균을 다시 결측치에 넣는 것이므로 중앙값 대체가 더 안전한 선택이다 | 다른 원인은 검토하지 않음

2. customer_age = 999(3건) 0(2건)
1보다 낮거나 110보다 큰 경우를 NaN 처리후 중앙값으로 대체 | 999 또는 0은 정상 범주내의 나이가 아님 | 아마도 입력 오류라 더 확실히 알 수가 없음

3. price 결측 (MCAR이 가능하지만 카테고리 정보 활용)
카테고리별 중앙값으로 대체 | 카테고리별 가격이 다를 수 있고 비율이 0.3%로 낮음 | 표본이 매우 적음

4. quantity = 200(1건)
NaN 처리 후 중앙값으로 대체 | 다른 값들은 1~3사이에 분포한다. 따라서 이상치로 판단하였음 | 진짜 데이터일 경우 손실이 발생

5. amount = 50,000,000
amount_outlier 플래그 칼럼을 만들어 유지 (보류) | 성격이 다른 거래일 가능성이 있음, 클리핑을 했을 시 극도로 큰 값 때문에 값이 뭉개졌다. 플래그를 할 시 값이 변경되지 않았다 | 소매인지 도매인지 추가 분석 필요

## 4. 처리 후 검증
- 결측 0건(설계상 NaN 유지가 필요한 컬럼 제외)
- customer_age 범위: 5 ~ 60 (정상)
- amount_outlier 플래그 1건 보존

## 5. 후속 권고
- 카테고리 별 평균 amount, 총 매출 비중 등 어떤 카테고리가 매출에 견인하는지 파악
- 정제된 데이터에서 app과 web의 평균 amount, price, quantity 분포 파악
- amount_outlier 추가 분석 필요 (포함했을때와 포함하지 않았을때의 매출 지표를 분석하고 outlier가 전체 데이터를 얼마나 왜곡하는지)
- 추가 상관관계 분석(price, quantity, customer_age 등)
- 현재 order_id만 존재, customer_id가 존재한다면 고객의 빈도, 금액 등을 고려하여 재구매 고객인지 1회성고객인지 판단가능, 또한 재구매의 비율이 높다면 어느 카테고리에서 높은지, 평균적인 가격대는 어떤지, 어느 채널에서 더 높았는지 등을 파악 가능
